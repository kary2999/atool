## Short description (≤ 132 chars)


## Category

`Developer Tools`

---

## Detailed description

```
WS Inspector is a full-featured WebSocket debugging extension for Chrome.
Intercept every WebSocket frame on the current page — including compressed
binary frames — and explore them in a sleek dark-themed three-column panel,
with no proxy or MITM setup required.

═══════ WHAT'S NEW IN 1.2 ═══════

✦ Dark hacker-theme UI — three-column layout inspired by Postman & Charles
  (Icon Nav | Message List | Detail Viewer), single-row toolbar
✦ WSS Diagnostics tab — check whether a WebSocket endpoint is reachable:
  DNS resolution → TLS → handshake, step-by-step timeline with timing
✦ Local send templates — save, load and delete payload presets in the Send tab
✦ Keyboard navigation — ↑ / ↓ moves through the message list,
  ← / → switches detail tabs (Decoded / Raw / Hex / Info)

═══════ CORE FEATURES ═══════

✓ Intercepts all WebSocket connections on the page (including dynamic ones)
✓ Auto-decompression — gzip / deflate / deflate-raw, magic-byte detection
✓ Four detail views: Decoded JSON (syntax highlighted) · Raw · Hex · Info
✓ Active connections — open ws:// or wss:// endpoints directly from the panel
✓ Heartbeat keep-alive + auto-reconnect with exponential back-off
✓ Subscription rules — auto-send frames on connect (auth tokens, subscribe payloads)
✓ Message color rules — keyword / regex / JSON-path based highlighting
✓ Pin important messages — stays at top regardless of scroll
✓ Direction filter — show only sent ↑ or received ↓
✓ Send templates (global) — save common payloads, send in one click
✓ Right-click context menu — copy / resend / pin / delete
✓ JSON export — archive the full filtered message log
✓ Live message counter with badge

═══════ TWO DISPLAY MODES ═══════

① DevTools Panel  (F12 → "WS Inspector" tab)
   Embedded alongside Chrome DevTools Network / Console panels

② Browser Side Panel  (click toolbar icon)
   Docked to the right of the page, auto-follows tab switches,
   clears automatically on navigation — no DevTools required

═══════ WHO IS THIS FOR ═══════

• Frontend / full-stack engineers debugging WebSocket APIs,
  live data feeds, or real-time chat
• QA engineers verifying push message payloads and server behavior
• Protocol analysts inspecting game clients, market-data feeds,
  or IM application traffic
• Anyone who needs to quickly test whether a WSS endpoint is reachable
  before writing a single line of client code

═══════ PRIVACY ═══════

• 100 % local — no WebSocket data is uploaded anywhere
• No user information collected or stored remotely
• Messages live only in memory and are released when the panel closes
• Fully open source and auditable

═══════ HOW TO USE ═══════

1. Install the extension
2. Open any page that uses WebSocket (trading platforms, chat apps, games…)
3. Click the toolbar icon for the Side Panel, or press F12 → WS Inspector tab
4. Reload the page — frames appear automatically, grouped by connection
5. Click any message to inspect; use ↑↓ to navigate, ←→ to switch tabs
6. Click "+ New Connection" to open your own WebSocket and send messages
7. Use the ⚡ Diagnostics tab to check if a WSS endpoint is reachable
```
